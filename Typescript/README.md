# TypeScript


### TypeScript란 무엇인가?

변수, 함수 매개변수, 함수 리턴값에 타입이 명시적으로 붙은 게 TypeScript 이다.
TypeScript에서 Type을 쓰지 않으면 Javascriopt랑 다른 게 없다...!

#### TypeScript를 사용하여 얻는 이점

TypeScript를 통해 약한 타입에서 강한 타입으로 변하면 다음과 같은 이점이 생긴다!
1. 안정적이다.
1. 에러가 적게 발생한다.
1. **에디터의 지원을 받을 수 있게 된다.**

가장 중요한 건 에디터의 지원을 받을 수 있다는 점이다. 

원래 약한 타입 언어인 Javascript를 개발할 때는 WebStorm이라는 IED가 높은 생산성을 제공해주고 에러날 상황도 미리 방지해주었는데..! TypeScript가 생기면서 VSCode로도 지원을 받을 수 있게 되었다. 

TypeScript와 VSCode 모두 MS(마이크로소프트)에서 만든 까닭에 둘은 호환이 잘 된다. 따라서 이러한 에디터 없이 타입스크립트를 하면 TypeScript를 사용할 때 얻는 이점이 다 사라진다. 타입을 외워서 감으로 한다는 것(머리속으로 한다는 것)은 js에서 ts로 전환하는 생산성이 떨어진다. 

### TypeScript의 중요한 개념

TypeScript는 `Compile(컴파일)`이라는 개념이 나온다. 
TypeScript는 언어이자 라이브러리이다. 
언어는 실행기가 있어야 한다. 

Javascript는 대표적으로 브라우저와 Node.js 라는 실행기가 있다. 
> 짚고 넘어가자면 Node.js는 서버가 아니라 자바스크립트 실행기이다. 브라우저도 메인 기능은 아니지만.. Javascript 엔진이 있으므로 실행기이다. 


타입스크립트의 실행기는 무엇인가?
디노???는 안정적인 버전이 아니다. 
타입스크립트의 실행기는 라이브러리이다. 



#### TypeScript를 실행하기 위해선 Node.js가 필요하다. 

TypeScript를 실행하기 위해선 Node.js가 필요하므로 어느 정도 Node에 대해 이해가 필요하다!

라이브러리를 설치해야 한다. 
타입스크립트를 컴파일해주는 라이브러리인 노드가 필요함...!
